# WeatherFlow Scheduler

WeatherFlow Scheduler — мой C++23/Qt проект для подбора мест рядом с пользователем по текущей погоде. Приложение определяет координаты, получает погодную сводку, выбирает подходящую стратегию поиска и показывает ближайшие места, куда сейчас разумно сходить.

Внутри проекта есть две основные части:

- `TTaskScheduler` — header-only библиотека для построения и выполнения графа зависимых задач.
- `places_recommender` — прикладной слой, который использует scheduler для цепочки сетевых запросов, парсинга и подготовки рекомендаций.

## Возможности

- определение координат по IP через 2ip;
- получение текущей погоды через Open-Meteo;
- поиск ближайших мест через Overpass API;
- подбор категорий мест под погодные условия;
- расчет расстояния до найденных мест по координатам;
- Qt-интерфейс с ручным вводом координат и кнопкой автоопределения;
- отдельные изображения для хорошей, нейтральной, дождливой, снежной и опасной погоды;
- тесты для библиотеки планировщика задач.

## Как это работает

Основная логика приложения описана как граф вычислений:

1. Получить координаты пользователя.
2. По координатам запросить текущую погоду.
3. Распарсить погодный ответ и классифицировать состояние погоды.
4. По погоде собрать Overpass-запрос с подходящими категориями мест.
5. Распарсить найденные объекты, посчитать расстояния и отсортировать их.
6. Передать результат в Qt-интерфейс.

Зависимости между шагами связывает `TTaskScheduler`: одни задачи отдают результат через `TFuture`, другие используют его как входной аргумент. Благодаря этому код приложения остается разделенным на понятные этапы, а scheduler отвечает за порядок выполнения.

## Логика рекомендаций

Приложение не просто ищет все подряд рядом с точкой. Перед поиском оно строит `SearchPlan`, который зависит от погоды:

- при дожде, высокой вероятности осадков или сильном ветре приоритет получают кафе, рестораны, кинотеатры, театры, библиотеки, музеи и торговые центры;
- при снеге и морозе радиус поиска уменьшается, а акцент смещается на закрытые места;
- в хорошую или жаркую погоду добавляются парки, сады, пляжи, обзорные точки и другие места для прогулки;
- при опасной погоде поиск становится более осторожным и ближним.

Расстояние считается локально по haversine-формуле, после чего места сортируются от ближайших к более дальним.

## TTaskScheduler

`TTaskScheduler` находится в `lib/` и подключается как interface-библиотека CMake. Он поддерживает:

- добавление задач через `add`;
- ленивое получение результата через `getResultSync`;
- выполнение всех задач через `executeAll`;
- передачу зависимостей через `getFutureResult`;
- цепочки обработки через `apply`;
- callable-объекты, лямбды и указатели на методы;
- move-only аргументы и результаты;
- разные режимы доступа к результату: перемещение значения или многократное чтение по ссылке.

Короткий пример:

```cpp
ts::TTaskScheduler scheduler;

auto temperature = scheduler.add([] {
   return 18.5f;
});

auto description = scheduler.add(
   [](float value) {
      return value > 15.0f ? "warm" : "cold";
   },
   temperature.getFutureResult<float>()
);

auto result = description.getResultSync();
```

## Зависимости

Для сборки нужны:

- CMake 3.20+;
- компилятор с поддержкой C++23;
- Qt6 Widgets;
- доступ к интернету при первой конфигурации CMake.

Через `FetchContent` автоматически подтягиваются:

- `cpr` для HTTP-запросов;
- `nlohmann/json` для JSON;
- `GoogleTest` для тестов.

Для работы приложения также нужен интернет: данные берутся из 2ip, Open-Meteo и Overpass API. API-ключи не требуются.

## Сборка и запуск

```bash
mkdir -p build
cd build
cmake ..
make
./weatherflow_scheduler
```

После запуска можно нажать `найти меня`, чтобы заполнить координаты автоматически, или ввести широту и долготу вручную. Кнопка `вычислить погоду и места` обновит погодную сводку, список рекомендаций и изображение состояния погоды.

## Тесты

Тесты покрывают поведение `TTaskScheduler`: выполнение задач, зависимости, `TFuture`, `apply`, move-семантику, ссылки, void-задачи и исключения.

Запуск:

```bash
cd build
make
ctest
```

Можно запустить и сам бинарник тестов:

```bash
cd build
./tests/task_scheduler_tests
```

## Ограничения

- Координаты по IP могут быть неточными, поэтому ручной ввод координат оставлен как основной сценарий для проверки конкретного места.
- Overpass API иногда отвечает медленно или может временно ограничивать запросы.
- Качество рекомендаций зависит от полноты данных OpenStreetMap в выбранной области.
- Приложение выполняет сетевые запросы синхронно, поэтому во время ожидания ответа интерфейс может ненадолго замирать.

## Технический фокус

Главная техническая часть проекта — собственный `TTaskScheduler`, который описывает вычисления как граф зависимых задач. На приложении он используется в реальном сценарии: сетевые запросы, парсинг JSON, классификация погоды, построение поискового запроса, сортировка мест и передача результата в интерфейс.

Такой подход помогает разделить бизнес-логику на независимые шаги и явно описать зависимости между ними.
